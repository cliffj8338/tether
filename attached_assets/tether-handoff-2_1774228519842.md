## PUNCH LIST — Remaining issues for Replit

These were visible in screenshots during the session and were NOT resolved.

---

### 1. Two tile-in-tile issues still visible on mobile (PRIORITY)

Screenshots from the session showed two sections still rendering nested tiles after the CSS fix. The CSS attribute selector approach has limits — it only catches inline `style=` attributes, not elements that have been structurally nested inside cards at the HTML level.

**How to find them:** Load the page on mobile (or DevTools mobile emulation). Scroll through every section. Any card that appears to be "inset" inside another card with double padding/border is a nested tile. The pattern is always the same: a styled `<div>` that opens immediately after `</table></div>` without first closing the parent `.card`.

**How to fix each one:** Find the pattern, close the parent card first, then open the inner block as a sibling.

---

### 2. Tables still clipping on mobile in some sections

Some tables that do NOT have a `.sx` wrapper are clipping horizontally on mobile instead of scrolling. They are inside cards with `overflow: hidden`.

**Fix:** Wrap those bare tables in `<div class="sx">...</div>`.

**How to find them:** Search for `<table class=` and check if the immediate parent is `.card` or `.sx`. Any `<table>` whose parent is `.card` (not `.sx`) will clip.

```bash
grep -n "<table class=" index.html
```

For each one, check if the line above is `<div class="sx">`. If not, wrap it.

---

### 3. Mobile: `.mkt-channel` cards have inconsistent left-edge alignment

The GTM section marketing channel cards (`.mkt-channel`) have a padding mismatch on mobile causing uneven left edges. Likely a `padding-left` or `margin-left` inconsistency in the `.mkt-channel` class at mobile breakpoint.

**Fix:** Add to `@media (max-width: 720px)`:
```css
.mkt-channel { padding: 16px; }
```

### 4. Mobile: `.g2` tiles have excess vertical gap in some sections

Some `.g2` grids that collapse to single column on mobile retain `gap: 24px` between items — looks like excessive whitespace. Tighten gap on mobile.

**Fix:** Add to `@media (max-width: 720px)`:
```css
.g2 { gap: 14px !important; }
```

### 5. Investor View toggle state not persisted on refresh

Expected behavior — documented but worth noting for future. When investor shares a link, recipient lands in Parent View. Could be solved with a `?investor=true` URL param check on DOMContentLoaded.
